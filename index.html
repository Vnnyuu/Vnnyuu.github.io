机器学习03——决策树

2017-04-29

简述

决策树是基于树结构 来进行决策的，就像带有终止块的流程图，终止块表示分类结果。

– 决策树
优点 计算复杂度不高，输出结果易于理解。对中间值缺失不敏感，可以处理不相关特征数据
缺点 可能会产生过度匹配问题
适用 数值型、标称型

原理

一颗决策树包含一个根结点、若干内部结点、若干叶子结点。
叶结点对应于决策结果，其他每个结点对应一个属性测试；
每个结点包含的样本集合根据属性测试的结果被划分到子结点中；
根结点包含样本全集。
从根结点到每个叶结点的路径对应了一个判定测试序列。
决策树学习的目的是为了产生一颗泛化能力强，即处理未见示例能力强的决策树。

信息增益（information gain）

划分数据集之前之后信息发生的变化称为信息增益。
信息增益最高的特征就是最好的选择。
集合信息的度量方式为香浓熵（简称 熵 entropy）。

公式：

如果待分类的事物可能划分在多个分类中，定义符号\( p(x_i) \)为选择该分类的概率

  • 公式1:\(l(x_i) = -log_2 p(x_i)\)

  • 公式2:\(H = -\sum_{i=1}^n p(x_i)log_2p(x_i)\)

计算熵值要计算所有类别所有可能值包含的信息期望值（n是分类的数目）

机器学习02——kNN

2017-04-28

简述

k-近邻算法（kNN,k-Nearest Neighbor）
采用测量不同特征值之间的距离方法进行分类。

– kNN
优点 精度高、对异常值不敏感、无输入数据假定
缺点 计算复杂度、空间复杂度高
适用 数值型、标称型

原理

存在一个样本数据集合（即训练样本集），并且样本集中每个数据都存在标签，即我们知道样本集中每一数据与所属分类的对应关系。
输入没有标签的新数据后，将新数据的每个特征与样本集中数据对应的特征进行比较，然后算法提取样本集中特征最相似数据（即最近邻）的分类标签。
一般来说，我们只选择样本数据集中前k个最相似的数据，这就是kNN的k。通常k是不大于20的整数。
最后，选择k个最相似数据中出现次数最多的分类，作为新数据的分类。

算法步骤

对未知类别属性的数据集中的每个点依次执行以下操作：
1）计算已知类别数据集中的点与当前点之间的距离。
2）按照距离递增次序排序。
3）选取与当前点距离最小的k个点。
4）确定前k个点所在类别的出现频率。
5）返回前k个点出现频率最高的类别作为当前点的预测分类。

总结

  • 依赖实例，必须有接近实际数据的训练集。
  • 必须保存全部数据集，占用存储空间。
  • 要对数据集中每个数据计算距离，耗时。
  • 无法给出任何数据的基础结构信息（内在含义），导致无法知晓平均实例样本和典型实例样本具有什么特征。

机器学习01——综述

2017-04-26

机器学习综述

监督学习

这类算法必须知道预测什么，即目标变量的分类信息。
我们给 学习算法 一个数据集（正样本），这个数据集由“正确答案”组成，机器可以从中推演出制定目标变量的可能结果。

监督学习 描述
分类问题 将实例数据划分到合适的分类中。例如：肿瘤检测问题。
回归问题 预测数值型数据。例如：房价问题。

监督学习算法

算法 描述（学到补充）
K-近邻
朴素贝叶斯
支持向量机
决策树
线性回归
局部加权线性回归
Ridge回归
Lasso最小回归系数估计

无监督学习

数据没有类别信息。也不会给定目标值。

无监督学习 描述
聚类问题 将数据集合分成由类似的对象组成的多个类的过程。
密度估计 寻找描述数据统计值的过程。
降维 减少数据特征的维度。

无监督学习算法

算法 描述（学到补充）
K-均值
最大期望值算法
DBSCAN
Parzen窗设计

算法如何选择

先看是否需要预测目标变量值

需要：监督学习

目标变量类型 算法类型
离散型 分类算法
连续型 回归算法

不需要：无监督学习

算法类型 应用
聚类算法 只需将数据划分为离散的组
密度估计算法 还需估计数据与每个分组的相似程度

解决问题的步骤

步骤 描述
1.收集数据 收集数据样本。
2.处理数据 确保数据格式符合要求，为算法准备特定的数据格式。
3.分析输入数据 确保数据集中没有垃圾数据。
4.训练算法 跑算法，将前三步得到的数据输入到算法中，从中抽取知识或信息。无监督学习没有前三步。
5.测试算法 实际使用前四步得到的知识信息。测试算法的工作效果。如果效果不尽人意，需要调到前几步进行。
6.使用算法 将算法实用化。

排序算法总结

2017-04-07

分类：

类别 名称
交换排序 冒泡排序、快速排序
插入排序 直接插入排序、希尔排序
选择排序 简单选择排序、堆排序
归并排序 归并排序

交换排序(冒泡排序、快速排序)

冒泡排序

最好 平均 最坏 稳定性
O(n) O(n^2) O(n^2) 稳定

从头开始两两比较，如果前面比后面大就交换，让较大的数向下沉，较小的数向上冒。
改进: 设置一个flag,检测这一趟有没有发生交换，如果没发生交换说明已经排序完毕了。

快速排序

最好 平均 最坏 稳定性
O(nlog2n) O(nlog2n) O(n^2) 不稳定

快速排序是一个不稳定的方法。
如果初始状态下有序或基本有序，快速排序其实就是冒泡排序。
但当初始状态是随机分布时，快速排序的平均时间最短。

1.先选出一个基准数
2.第一次排列，比基准数小的在基准数的前面，比基准数大的放在基准数的后面（分区操作）
3.被分的区再次进行步骤1.2，直到有序。

插入排序(直接插入排序、希尔排序)

直接插入排序

最好 平均 最坏 稳定性
O(n) O(n^2) O(n^2) 稳定

在少量数据下是个不错的选择，在STL的sort算法和stdlib的qsort算法中，都将插入排序作为快速排序的补充，用于少量元素的排序(<=8)

1.取第一个元素
2.取下一个元素，在已排序的序列中从后向前扫描
3.如果序列中的某元素大于新元素，则后移一位
4.重复3，直到找到序列中的元素小于或等于新元素
5.将新元素插入到该元素后面
6.重复2~5步，直到序列有序

希尔排序

最好 平均 最坏 稳定性
O(n) O(n^1.3) O(n^2) 不稳定

1.选取一个小于序列长度的数作为第一个增量，将序列分成若干组。
2.每组数内部进行排序。
3.取小于第一个增量的第二个增量，重复。
4.直到所取的增量为1，进行一次直接插入排序。

选择排序(简单选择排序、堆排序)

简单选择排序

最好 平均 最坏 稳定性
O(n^2) O(n^2) O(n^2) 不稳定

在要排序的一组数中，选出最小的一个数与第1个位置的数交换；然后在剩下的数当中再找最小的与第2个位置的数交换，依次类推，直到第n-1个元素和第n个元素比较为止。

堆排序

最好 平均 最坏 稳定性
O(nlog2n) O(nlog2n) O(nlog2n) 不稳定

堆排序是一种树形选择排序，是对直接选择排序的有效改进。
堆顶元素（第一个元素）必为最小项。
堆排序需要两个过程：建堆 和 输出堆顶后剩下元素的调整。

归并排序

最好 平均 最坏 稳定性
O(nlog2n) O(nlog2n) O(nlog2n) 稳定

是分治法的应用。
将两个（及以上）有序表合并成一个新的有序表，即把待排序序列分为若干个子序列，每个子序列是有序的。然后再把有序子序列合并为整体有序序列。
